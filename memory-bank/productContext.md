# Product Context

## Sound Board - AI Interview Assistant

**Version:** 0.4.0
**Last Updated:** September 29, 2025

## Problem Statement

Technical interviews are challenging and stressful for many developers. Candidates often struggle with:
- Time pressure and performance anxiety
- Complex algorithmic problems they haven't encountered before
- Need to explain their thought process while coding
- Limited access to real-time assistance during interviews

## Solution Overview

Sound Board addresses these challenges by providing an AI-powered assistant that:
- Listens to the interview conversation in real-time
- Analyzes screen content to understand the context
- Provides helpful, context-aware responses without being detected
- Maintains ethical boundaries by focusing on learning and understanding rather than cheating

## User Experience Goals

### Primary User Journey
1. **Setup**: User configures API keys and preferences
2. **Start Session**: User begins interview session with audio/screen capture
3. **Real-time Assistance**: AI provides helpful responses based on context
4. **Review**: User can review conversation history for learning

## Ethical Considerations

### Responsible AI Usage
- **Transparency**: Clear that this is an AI assistant tool
- **Boundaries**: Focus on learning and understanding, not cheating
- **User Control**: Users control when and how they use the tool
- **Educational Value**: Emphasis on skill development over shortcuts

### Privacy and Security
- **Local Processing**: Minimal data sent to external services
- **User Control**: Users control their API keys and data
- **No Data Storage**: Conversations stored locally only
- **Content Protection**: Stealth mode prevents detection

## Competitive Landscape

### Direct Competitors
- **Interviewing.io**: Live practice with real engineers

### Indirect Competitors
- **ChatGPT/Claude**: General AI assistants

### Differentiation
- **Real-time Context**: Listens to actual interview conversations
- **Screen Integration**: Understands visual context from screenshots
- **Stealth Mode**: Designed for undetectable assistance
- **Multi-LLM Support**: Choice of different AI providers

## Technical Constraints

### Platform Limitations
- **Audio Capture**: Different APIs on Windows/macOS/Linux
- **Screen Recording**: Privacy restrictions and performance concerns
- **Electron Overhead**: Desktop app performance considerations
- **Network Dependency**: Requires internet for LLM APIs

### User Constraints
- **API Costs**: Users must pay for their own LLM usage
- **Technical Setup**: Requires API key configuration
- **Privacy Concerns**: Some users may be wary of screen capture
- **Learning Curve**: Understanding how to use the tool effectively
